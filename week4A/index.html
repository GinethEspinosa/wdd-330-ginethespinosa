Ch11: Further Functions

  1. Functions have built-in properties such as length , but can have custom properties added.
  2. All functions have call() and apply() methods that can invoke a function with the value of this bound to an object that is provided as an argument.
  3. Immediately Invoked Function Expressions or IIFEs are functions that are enclosed in parentheses and immediately followed by double parentheses so they’re invoked. They are useful for namespacing variables and setting default values.
  4. Functions are able to dynamically redefine themselves in the body of the function, depending on certain conditions.
  5. A recursive function will keep invoking itself until a certain condition is met.
  6. A callback is a function that’s provided as an argument to another function.
  7. Callbacks are frequently used in asynchronous programming as part of the event loop. This means that a program can continue to run in a single thread while waiting for another task to be completed.
  8. Promises can be used instead of callbacks to deal with multiple asynchronous actions in sequence. They also provide a nicer mechanism for handling errors.
  9. Functions that return other functions are known as higher-order functions.
  10. A closure is the process of keeping a reference to a variable available outside the scope of the function it was originally defined in.
  11. A generator is created by placing an asterisk character (*) after the function keyword.
  12. A generator function will return an iterator object that provides a next() method, which returns the next value in a sequence that is defined in the generator function.
  13. Functional programming involves breaking processes down into steps that can be applied as a series of functions.
  14. Pure functions are functions that don't rely on the state of the code they are called from, have no side-effects, and always give the same result when given the same arguments (referential transparency).
  15. Currying or partial application is the process of applying one argument at a time to a function. A new function is returned until all the arguments have been used.

Ch8: Transforms and Transitions

  1. CSS3: without a line of JavaScript or a single JPEG, you can tilt, scale, move, and even flip your elements with ease.
  2. Transforms: The CSS3 transform property lets you lets you translate, rotate, scale, and/or skew any element on the page.
  3. Translation: Translation functions allow you to move elements left, right, up, or down. The translate(x,y) function moves an element x from the left, and y from the top.
  4. Scaling: The scale(x,y) function scales an element by the defined factors horizontally then vertically.
  5. Transitions allow the values of CSS properties to change over time, essentially providing simple animations.
  6. Rotation: The rotate() function rotates an element around the point of origin by a specified angle value.
  7. Skew: The skew(x,y) function specifies a skew along the x and y axes.
  8. Animations: You can define starting and ending states, but there’s no fine-grained control over any intermediate states.
  9. Keyframes: To animate an element in CSS, you first create a named animation, then attach it to an element in that element’s property declaration block. Animations in themselves don’t do anything; in order to animate an element, you’ll need to associate the animation with that element.
  10. Use animations and transitions where it makes sense, enhancing the user experience—and skip it everywhere else.
